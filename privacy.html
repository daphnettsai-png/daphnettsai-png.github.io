Privacy Decision Tool

Developed in collaboration with Edie Cahill and Julie Song.

This project is an interactive decision-tree tool that helps engineers choose privacy-preserving technologies based on the kind of data they are handling, the sensitivity of the use case, and the level of trust in the system.

Open Interactive Quiz Back to Portfolio

Project Overview

The goal of this tool is to make privacy engineering more understandable for people who may not have a privacy or security background. Instead of giving one universal answer, the tool guides users through questions about their specific task and recommends a privacy technology that fits that use case.

Problem

Engineers often know they need to protect user data, but it can be hard to know whether they should use anonymization, differential privacy, cryptography, or trusted execution environments.

Solution

We designed a decision-tree framework and quiz that asks practical questions about trust, data sensitivity, and output type to guide users toward an appropriate privacy-preserving technology.

Technologies Covered

Differential Privacy

Useful for aggregate analysis when the goal is to learn patterns from a dataset while limiting how much any one person’s data can affect the result.

Anonymization

Useful when identifying details are removed or generalized, though it may not be enough by itself for stronger privacy guarantees.

Cryptography

Useful when information needs to be protected during storage or transmission so unauthorized people cannot read it.

Trusted Execution Environments

Useful when sensitive data must be processed in a protected hardware-backed environment with stronger guarantees around isolation.

How the Tool Works

  • Starts with questions about the kind of sensitive task being performed.
  • Considers whether there is a trusted central party.
  • Looks at whether the output is aggregate, encrypted, anonymized, or processed in a protected environment.
  • Leads users to a recommended privacy technology with supporting explanation.
